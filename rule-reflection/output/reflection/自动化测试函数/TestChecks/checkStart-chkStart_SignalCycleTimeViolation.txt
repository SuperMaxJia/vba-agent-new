##功能##
该函数用于启动信号周期时间违规检查，监控指定信号是否在预期周期时间内未更新或更新频率异常。主要用于自动化测试中验证总线信号的一致性，确保信号按预设周期正常更新。

##vba规范##
函数名：chkStart_SignalCycleTimeViolation  
参数：signalName (String类型) - 要监控的信号名称  
返回值：无  
依赖：需先调用chkCreate_SignalCycleTimeViolation创建检查规则  
行为：启动对指定信号的周期时间监控，违反规则时记录违规事件  

##capl规范##
函数名：checkStart  
参数：char signalName[] - 要监控的信号名称  
返回值：void  
依赖：需先调用checkCreate设置监控规则  
示例1：  
```capl
// 启动对EngineSpeed信号的周期检查
checkStart("EngineSpeed");
```

示例2：  
```capl
// 启动多信号检查
checkStart("VehicleSpeed");
checkStart("EngineTemp");
```

##capl名##
checkStart

##转换须知##
1. CAPL使用char数组表示字符串，VBA使用String类型
2. CAPL函数无返回值，VBA函数同样无返回值
3. CAPL中信号名需用双引号包裹，VBA中同样
4. 两者都需要先创建检查规则再启动监控
5. CAPL函数名更简洁，VBA采用更详细的命名约定

##转换示例##
CAPL原始代码：
```capl
// 启动灯光信号检查
checkStart("HeadLightStatus");
```

转换后的VBA代码：
```vba
' 启动灯光信号检查
chkStart_SignalCycleTimeViolation "HeadLightStatus"
```

关键转换步骤：
1. 函数名从checkStart改为chkStart_SignalCycleTimeViolation
2. 保持相同的信号名参数传递方式
3. 移除CAPL的分号结尾
4. 添加VBA注释前缀
