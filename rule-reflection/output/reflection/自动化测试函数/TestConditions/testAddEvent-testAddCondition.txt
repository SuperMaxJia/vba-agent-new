##功能##
用于添加一个事件对象或者事件文本作为测试的条件，将对象的统计信息同步到自动化测试报告中。behavior参数控制是否改变测试用例状态。

##vba规范##
函数定义：testAddCondition(object, behavior = 0)
参数：
- object：事件对象
- behavior：int类型，默认为0。0表示改变测试状态，1表示仅记录不改变状态
返回值：int类型，0表示成功，-1表示错误

##capl规范##
函数定义：testAddEvent(event, behavior = 0)
参数：
- event：事件对象或文本
- behavior：int类型，默认为0
返回值：int类型，0表示成功，-1表示错误

示例1：
```capl
on key 'a'
{
  int result;
  result = testAddEvent(this, 0);
  write("Add event result: %d", result);
}
```

示例2：
```capl
int addCondition()
{
  return testAddEvent("EngineStartEvent", 1);
}
```

##capl名##
testAddEvent

##转换须知##
1. CAPL使用event参数名而非object
2. CAPL支持直接传入事件文本字符串
3. 返回值类型和含义保持一致
4. 默认参数值相同

##转换示例##
1. CAPL原始代码：
```capl
int res = testAddEvent(canMessage, 1);
```

2. 转换后的VBA代码：
```vba
Dim res As Integer
res = testAddCondition(canMessage, 1)
```

3. 关键转换说明：
- 函数名从testAddEvent改为testAddCondition
- 参数名保持不变
- 返回值处理方式相同
- 参数默认值逻辑一致
