1. 函数名
testAddTriggerTesterAction

2. 函数描述
用于创建一个测试触发，当条件被触发时调用该函数。

3. 函数体
testAddTriggerTesterAction(acition, callbackTrigger)
或者
testAddTriggerTesterAction(acition, callbackCycle)

4. 返回值
无明确返回值说明（根据示例推断可能返回触发ID）

5. 参数
| 名称           | 类型                | 描述                                                                 |
|----------------|---------------------|----------------------------------------------------------------------|
| acition        | testCreateTesterAction | testCreateTesterAction描述中的测试动作对象                          |
| callbackTrigger| str                 | 触发条件，可以在信号或者系统变量再发生变化的时候触发                 |
| callbackCycle  | int                 | 触发的周期                                                          |

6. 内置函数
无特定内置函数说明

7. 示例
示例功能：演示如何创建不同类型的测试触发（周期触发、信号触发、变量触发）

示例代码：
gTriggerCycle = -1
gTriggerSig = -1
gTriggerSV = -1

def CallbackTesterAction(trigger):
    if (trigger == gTriggerCycle):
        testStep("", "Triggered by cycle time");
    elif (trigger == gTriggerSig):
        testStep("", "Triggered by signal");
    elif (trigger == gTriggerSV):
        testStep("", "Triggered by system variable");
    elif (trigger == gTriggerEV):
        testStep("", "Triggered by environment variable");
    else:
        testStep("", "Triggered before opening dialog or before closing dialog after timeout");
    if (sysVarEngineStarted == 1):
        return 1;
    return 0;

@testcase
def TCTesterAction():
    global gTriggerCycle, gTriggerSig, gTriggerSV
    # 创建一个handle，并绑定action对应的触发函数
    handle = testCreateTesterAction('This is the tester action text', 'Tester action heading', CallbackTesterAction);
    
    # 创建一个周期触发方式，action对应的触发函数将按周期触发。
    gTriggerCycle = TestAddTriggerTesterAction(handle, 1000);
    
    # 创建一个信号触发方式，action对应的触发函数将在信号改变时触发。
    gTriggerSig = TestAddTriggerTesterAction(handle, CAN.MW.SigSigned16);
    
    # 创建一个变量触发方式，action对应的触发函数将在变量值改变时触发。
    gTriggerSV = TestAddTriggerTesterAction(handle, sysvar.sysInt);
    
    # 创运行该函数后将打开一个弹窗，以上任一条件触发时，会自动关闭该窗口；窗口也可以在超时或者手动进行关闭，并将内容输出到测试报告。
    testValidateTesterAction(handle, 10000, 1);

8. 其他说明
- 该函数通常与testCreateTesterAction和testValidateTesterAction配合使用
- 可以创建多种类型的触发方式（周期、信号变化、变量变化等）
- 触发时会调用指定的回调函数，并通过参数传递触发源标识
- 示例中展示了全局变量用于存储不同触发类型的ID
