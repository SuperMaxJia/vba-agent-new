1. 函数名  
chkCreate_MsgRelCycleTimeViolation  

2. 函数描述  
用于检测数据库中定义的周期性报文的周期是否在指定的范围内。  

3. 函数体  
所有检测函数都有对应的chkStart_xxxx()和chkCreate_xxxx()。  
chkCreate_xxxx()只创建检测对象但不开始检测，需要调用chkControl_Start；  
chkStart_xxxx()创建检测对象同时直接开始检测。  

4. 返回值  
名称 | 类型 | 描述  
res | int | 代表创建的对象，-1代表创建失败  

5. 参数  
名称 | 类型 | 描述  
message | object | 在数据库中定义的报文对象，基于数据库中定义的默认周期进行对比计算  
aMinRelCycleTime | Number | 0 < x < 1  
aMaxRelCycleTime | Number | 1 < x < ∞  

6. 内置函数  
函数名称 | 函数描述 | 参数说明  
chkControl_Start | 启动检测对象 | 无  
testAddCondition | 将统计信息加入自动化测试报告 | checkId（检测对象ID）  
testWaitForTimeout | 设置统计时长 | 时长（毫秒）  
testRemoveCondition | 结束统计 | checkId（检测对象ID）  

7. 示例  
示例功能：检测数据库中定义的周期性报文的周期是否在指定的范围内。  
示例代码：  
```vba  
#假如该报文在数据库中规定的周期为100ms，则以下函数检测该报文的实际周期是否在100*0.9到100*1.1的范围内。  
checkId = chkStart_MsgRelCycleTimeViolation(CAN1.MsgToObserve, 0.9, 1.1);  
#调用该方法后，统计信息将加入到自动化测试报告  
testAddCondition(checkId);  
#共统计1000ms  
testWaitForTimeout(1000);  
#结束统计  
testRemoveCondition(checkId);  
```  
代码注释：  
- `chkStart_MsgRelCycleTimeViolation` 创建并启动检测对象，参数为报文对象、最小相对周期（0.9）、最大相对周期（1.1）。  
- `testAddCondition` 将检测结果加入测试报告。  
- `testWaitForTimeout` 设置统计时长为1000毫秒。  
- `testRemoveCondition` 结束统计并移除检测对象。  

8. 其他和函数或变量强相关的内容  
- 该函数属于自动化测试函数中的TestChecks类别。  
- 需要配合其他测试控制函数（如`chkControl_Start`）使用以实现完整功能。  
- 适用于周期性报文监控场景，确保报文周期符合预期范围。
